---

| Property | Value |
|----------|-------|

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge offsets, inline padding |
| sm | 8px | Compact element spacing, stat card internal gaps |
| md | 16px | Default element spacing, card padding on mobile |
| lg | 24px | Section padding, card content padding on desktop |
| xl | 32px | Layout gaps between bento grid cells |

| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Touch targets for Edit Profile button and avatar hover overlay must be minimum 44px.

## Color

Uses existing Senkou theme tokens from `src/styles.css`. All values reference CSS custom properties, not raw hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / slate-1 | Page background, hero banner background |

## Interaction Contract

### View Mode (Default)

| Interaction | Behavior |
|-------------|----------|
| Page load | Cards animate in with `animate-fade-in` (existing keyframe, staggered 100ms per card using `animation-delay`) |
| Avatar hover | `scale-105` over 200ms, primary ring glow appears |
| Avatar click | Opens full-size image in Dialog (if image exists) |
| Card hover (bento cells) | Subtle lift `-translate-y-0.5` + shadow-elevated, 200ms ease |
| Edit button click | Opens edit Sheet (side drawer, right side) on desktop, full-screen Dialog on mobile (< 640px) |
| Stat card hover | Existing behavior: border brightens, shadow appears with status glow |

### Edit Mode (Sheet / Dialog)

| Interaction | Behavior |
|-------------|----------|
| Trigger | "Edit Profile" button in hero banner top-right area |
| Container | Sheet (from right, width 420px) on sm+, Dialog (centered) on mobile |
| Fields | Display Name (Input), Email (Input), Bio (Textarea with 128 char counter), Experience Level (custom slider with visual segment preview), Avatar (file upload with preview) |
| Experience slider | Custom styled: 10 discrete stops, shows emoji + title + subtitle updating live as user drags. Current level segment pulses gently. |
| Avatar upload | Click current avatar preview (80px circle) to trigger file input. Shows new image immediately as preview. "Remove Avatar" ghost button below if image exists. |
| Avatar removal | Clicking "Remove Avatar" shows an undo toast: "Avatar removed -- Undo" (with 5-second undo window). Avatar preview immediately switches to letter fallback. If undo is clicked, avatar is restored. If undo expires, removal is committed on next save. |
| Bio counter | Right-aligned `{count}/128`, color shifts to `text-destructive` at 120+ characters |
| Save | "Save Changes" primary button, full-width at bottom of sheet. Shows spinner icon during mutation. Disabled when no changes detected. |
| Discard | "Discard Changes" ghost button next to save. Also: clicking Sheet overlay or pressing Escape. |
| Success | Sheet closes, toast appears: "Profile updated -- Your changes have been saved." |
| Error | Inline error message below save button in `text-destructive`. Toast also fires. |
| Validation | Client-side: Display name required, email required + format check. Bio max 128. File max 256KB. |

### Loading States

| State | Visual |
|-------|--------|
| Initial profile load | Avatar: 96px pulsing circle `bg-muted/40`. Name: 160px x 28px pulse bar. Bio: 200px x 14px pulse bar. Experience: 48px circle pulse + 120px bar pulse. Stat cards: existing StatCardSkeleton (5 instances). |
| Save in progress | Save button text changes to "Saving...", button disabled, spinner icon replaces Save icon. All form fields disabled. |
| Stats loading | Existing skeleton pattern from ProfileAnalytics (unchanged). |

### Empty States

| State | Visual |
|-------|--------|
| No bio | Italic muted text: "No bio yet -- tell the world about your anime taste." |
| No avatar | Gradient circle (from-primary to-primary/80) with first letter of display name, 96px, 36px bold letter |
| No anime stats (total = 0) | Existing empty state from ProfileAnalytics: BookMarked icon 48px muted, "No anime in your library yet." + "Start adding anime to track your journey!" |
| Experience level 0 | Full experience card shown normally. All 10 segments unfilled except first. Label: "Episode Zero -- Just starting out" |

## Animation Contract

| Animation | Keyframe | Duration | Easing | Trigger |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Avatar image | `alt="Profile photo of {displayName}"` or `alt="Profile"` if no name |
| Avatar expand icon | Maximize2 icon-only button: `aria-label="View full size"` and visible tooltip on hover |
| Experience badge | Existing `role="img"` + `aria-label` pattern (unchanged) |
| Edit sheet | `aria-label="Edit profile"` on Sheet/Dialog. Focus trap handled by Radix. |
| Form fields | All inputs have associated `<Label htmlFor>` |
| Save button loading | `aria-busy="true"` + `aria-disabled="true"` during mutation |
| Stat cards | Each stat card is a `<div>` (non-interactive). Screen reader sees label + number naturally. |
| Progress bars | `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax` on experience bar and stat bars |
| Color contrast | All text meets WCAG 2.1 AA minimum. Yellow-9 on slate-1 background passes (verified by Radix color scale design). Muted-foreground (gray-9) on slate-1 passes. |
| Keyboard | Tab order: Edit button -> (within sheet) form fields in order -> Save -> Discard Changes. Escape closes sheet. |
| Reduced motion | Wrap all custom animations in `@media (prefers-reduced-motion: no-preference)`. Cards render without stagger/lift when motion is reduced. |
